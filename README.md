# Air Quality Prediction and Health Recommendation System

## Project structure

- `dataset/`
  - `air_quality_india.csv` (input dataset)
- `model/`
  - `train_model.py` (training script)
  - `model.pkl` (saved model, generated after script run)
  - `metrics.json` (model performance + feature importance)
  - `feature_importance.png` (plot generated by training script)
- `backend/`
  - `app.py` (Flask API)
- `frontend/`
  - `index.html` (UI)
  - `styles.css`
  - `script.js`
- `requirements.txt` (Python dependencies)

## 1) Setup

1. Open terminal in project root (`c:\Users\Win\Desktop\ML_PROJECT`).
2. Create virtual environment and activate:

```powershell
python -m venv venv
venv\Scripts\Activate.ps1
```

3. Install dependencies:

```powershell
pip install --upgrade pip
pip install -r requirements.txt
```

## 2) Prepare dataset and train model

1. Put the Kaggle dataset `Air Quality Data in India (2015–2020)` CSV into `dataset/air_quality_india.csv`.
2. Run training script:

```powershell
python model/train_model.py
```

- This script handles missing values, trains Linear Regression and Random Forest, evaluates R2 & MSE, selects best model, saves `model/model.pkl`, and writes `model/metrics.json`.

## 3) Run backend API

```powershell
python backend/app.py
```

- API runs at `http://127.0.0.1:5000`
- `/predict` POST with JSON body containing `pm25`, `pm10`, `no2`, `so2`, `co`.
- `/metrics` returns model comparison and feature importance.

## 4) Run frontend

Using a simple static server (recommended):

```powershell
cd frontend
python -m http.server 8000
```

Open browser: `http://127.0.0.1:8000`

## 5) Example API request

```powershell
curl -X POST "http://127.0.0.1:5000/predict" -H "Content-Type: application/json" -d "{\"pm25\": 45, \"pm10\": 70, \"no2\": 28, \"so2\": 8, \"co\": 0.8}"
```

## 6) VIVA support (short explanations)

- Machine learning: using data to teach a model patterns, so it can make predictions for new data.
- Regression: predicting a numeric output (like AQI) from input features (pollution values).
- Random Forest vs Linear Regression:
  - Linear Regression fits one straight line (simple, fast) and cannot handle complex non-linear relationships well.
  - Random Forest uses many decision trees and averages results; handles nonlinear data and interactions better.
- Training/testing split: put 80% data in training to learn model, 20% in testing to evaluate how well model generalizes to unseen data.
